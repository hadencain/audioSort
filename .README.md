# Audio File Organizer

A Python utility for organizing audio samples into structured categories based on filename keywords.

## Overview

Scans a source directory for audio files (WAV, MP3, AIF, AIFF, FLAC, OGG) and organizes them into categorized folders based on predefined keywords. It's particularly useful for organizing sample libraries, sound effects collections, or any large audio file repository.

## Features

- Intelligent Categorization Automatically sorts files based on keywords in filenames
- Duplicate Detection Prevents duplicate files using binary comparison
- Subcategory Support Special handling for drum samples with subcategories
- Name Conflict Resolution Automatically generates unique filenames when conflicts occur
- Detailed Logging Provides progress updates and statistics during operation

## Requirements

- Python 3.6 or higher
- Standard library modules only (no external dependencies)

## Installation

1. Clone or download this repository
2. Ensure you have a valid `categories.json` file in the same directory as the script

## Configuration

Create a `categories.json` file in the same directory as the script. This file defines the categories and keywords for file classification

```json
{
  drums {
    kicks [kick, bd, bass drum],
    snares [snare, sd, snr],
    hi-hats [hat, hh, hi hat, hihat],
    cymbals [crash, ride, cymbal, splash]
  },
  bass [bass, sub, 808],
  synths [synth, lead, pad, arp],
  fx [fx, effect, transition, riser, downlifter, impact],
  vocals [vocal, vox, acapella, voice]
}
```

Files that don't match any category will be placed in an others folder.

## How It Works

1. The script scans the source directory recursively for audio files
2. Each file is analyzed to determine its category based on the keywords in `categories.json`
3. Files are copied to the appropriate destination folder, preserving their original filenames
4. Duplicate detection prevents copying the same file multiple times
5. Name conflicts are resolved by appending a counter to the filename
